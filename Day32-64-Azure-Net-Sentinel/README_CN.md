# AI驱动的日志异常诊断系统：Oracle 26ai RAG + 混合搜索 + 信号驱动诊断

> **企业级RAG系统，结合混合检索（向量+BM25）、异常信号检测和重排序技术，实现智能日志诊断。支持动态文件上传、实时进度跟踪和批量处理优化。**

## 📌 项目概述

本项目是一个**生产级异常诊断系统**，帮助SRE从原始日志快速定位到**可解释的异常假设**，通过结合统计信号检测与基于LLM的模式推理，实现从噪声到信号的智能过滤。

> **📖 关于系统设计理念和问题陈述，请参阅 [SYSTEM_OVERVIEW.md](./SYSTEM_OVERVIEW.md)**

**核心特性：**

- **三阶段诊断流程**：信号检测 → 信号驱动检索 → RAG诊断
- **OpenAI文本嵌入** (text-embedding-3-large, 3072维)
- **Oracle Autonomous Database 26ai** 原生向量搜索
- **BM25关键词搜索** 用于精确词汇匹配
- **混合融合** (70% 向量 + 30% BM25)
- **CrossEncoder重排序** 提升检索精度
- **异常信号自动检测** 基于时间序列统计
- **动态文件上传** - 通过Web界面上传自定义日志文件
- **实时进度跟踪** - 监控处理进度，实时更新
- **批量处理优化** - 高效的批量嵌入生成（每批1000条）
- **FastAPI后端** RESTful API
- **现代化聊天式前端** 手风琴UI
- **完整测试覆盖** 单元测试、集成测试、API测试
- **评估框架** 支持信号检测准确率、噪声降低率等指标
- **CI/CD管道** 代码检查和测试自动化

用户可以通过以下方式使用系统：

1. **上传自定义日志文件** (.log, .txt, .gz) 最多5000条
2. **监控实时进度** 处理过程中查看进度
3. **执行异常诊断** 自动检测异常信号并生成诊断报告
4. **自然语言查询** 例如：
   - *"什么导致了块丢失？"*
   - *"为什么DataNode停止响应？"*
   - *"PacketResponder终止的原因"*

系统通过混合搜索检索**最相关的日志**，并基于检索到的证据生成**简洁的AI诊断**。

## 🚀 核心功能

### 1. 三阶段诊断流程

#### Stage 0: 异常信号检测
- **日志解析** 将日志解析为结构化格式（时间戳、级别、组件、消息）
- **模板提取** 使用Drain3提取消息模板（泛化动态部分：数字、IP、路径）
- **时间窗口聚合** 按5分钟窗口聚合日志
- **异常检测** 检测异常峰值：
  - 错误率 > 历史p95
  - 日志量变化 > 3倍
  - 新模板出现
- **异常评分** 错误优先加权（80%错误计数，20%量级峰值）
- **输出** 存储Top-3可疑时间窗口及其模板签名

#### Stage 1: 信号驱动检索
- **签名匹配** 使用`REGEXP_LIKE`在时间窗口内匹配日志签名
- **错误过滤** 过滤包含错误关键词的日志（ERROR, FATAL, Exception, 404, 500）
- **候选合并** 合并所有信号的候选日志，去重，限制为300条
- **输出** 高信噪比子集（通常100-300条日志）

#### Stage 2: RAG诊断层
- **统计分析** 对候选子集进行统计分析：
  - 错误类型分布
  - 时间集中模式
  - 常见关键词/组件
  - 消息模式频率
- **可选重排序** 如果提供用户查询，使用混合搜索重排序
- **LLM诊断** 将统计摘要+代表性样本输入LLM
- **结构化输出** 强制JSON格式：`root_cause`, `confidence`, `evidence`, `alternatives`, `next_steps`
- **输出** 可解释的诊断报告，包含基于模式的证据

### 2. 动态文件上传

- **基于Web的上传界面** - 拖放或点击选择文件

## 🛠️ 安装

### 1. 克隆仓库

```bash
git clone https://github.com/qiaonipan/90-days-ai-backend-reborn.git
cd Day27-30-production-rag-v2
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 配置环境变量

在项目根目录创建 `.env` 文件：

```env
# OpenAI API密钥
OPENAI_API_KEY=your_openai_api_key_here

# Oracle数据库配置
ORACLE_USERNAME=your_oracle_username
ORACLE_PASSWORD=your_oracle_password
ORACLE_DSN=your_oracle_dsn
ORACLE_WALLET_PATH=path_to_your_wallet_directory

# 可选：日志级别
LOG_LEVEL=INFO
```

### 4. 准备数据库表

确保Oracle数据库有以下表：

```sql
-- 文档表（存储日志和向量嵌入）
CREATE TABLE docs (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    text CLOB,
    embedding VECTOR(3072, FLOAT32),
    ts TIMESTAMP
);

-- 异常信号表（存储检测到的异常信号）
CREATE TABLE anomaly_signals (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    window_start TIMESTAMP,
    template_id NUMBER,
    signature VARCHAR2(4000),
    count NUMBER,
    score NUMBER
);
```

**⚠️ 重要**：如果从 `text-embedding-3-small` (1536维) 升级到 `text-embedding-3-large` (3072维)，需要修改表：

```sql
-- 删除现有表并重建（如果可以丢失现有数据）
DROP TABLE docs;
CREATE TABLE docs (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    text CLOB,
    embedding VECTOR(3072, FLOAT32),
    ts TIMESTAMP
);

-- 或者修改列（如果Oracle版本支持）
ALTER TABLE docs MODIFY embedding VECTOR(3072, FLOAT32);
```

## 🔧 技术细节

### 混合搜索算法

1. **向量搜索**：
   - 使用Oracle `VECTOR_DISTANCE` 函数
   - 返回Top 2×k候选
   - 将分数归一化到0-1范围

2. **BM25搜索**：
   - 对查询和语料库进行分词
   - 计算BM25分数
   - 返回Top 2×k候选

3. **融合**：
   - 向量分数 × 0.7 + BM25分数 × 0.3
   - 按融合分数排序
   - 返回Top k结果

4. **重排序**：
   - 使用CrossEncoder模型（`cross-encoder/ms-marco-MiniLM-L-6-v2`）
   - 计算查询-文档相关性分数
   - 按重排序分数重新排序
   - 返回Top k结果

### 相似度分数计算

- **公式**：`similarity = 1 - distance`
- **范围**：0.0 - 1.0（越高越好）
- **显示**：相关结果通常显示0.5 - 0.9

### 支持的日志格式

系统自动检测和解析多种日志格式：

1. **HDFS日志**（主要）：
   ```
   081110 145404 34 ERROR dfs.DataNode$PacketResponder: Received block blk_12345
   ```
   格式：`YYMMDD HHMMSS LEVEL COMPONENT: MESSAGE`

2. **Kubernetes日志**：
   - RFC3339时间戳格式：`2024-01-02T10:30:45.123Z INFO [component] message`
   - JSON格式：`{"timestamp":"2024-01-02T10:30:45Z","level":"ERROR","pod":"my-pod","namespace":"default","message":"..."}`
   - 自动提取Pod和Namespace元数据

3. **Nginx日志**：
   ```
   [02/Jan/2024:10:30:45] ERROR nginx: connection refused
   ```

4. **通用日志**：
   - ISO时间戳：`2024-01-02 10:30:45` 或 `2024-01-02T10:30:45`
   - 自动检测日志级别：ERROR, WARN, INFO, DEBUG, FATAL, CRITICAL
   - 从括号中提取组件：`[component]`

### 文件上传和处理

1. **文件上传**：
   - 接受 `.log`, `.txt`, 和 `.gz`（压缩）文件
   - 验证文件格式和大小
   - 解析日志条目（每个条目至少20个字符）
   - 限制为每个文件5000条
   - 自动格式检测和解析

2. **异常信号提取**：
   - 使用Drain3提取消息模板
   - 按5分钟窗口聚合
   - 检测错误率峰值、量级变化、新模板
   - 计算异常分数（80%错误，20%量级）
   - 存储Top-3异常信号

3. **批量嵌入生成**：
   - 将日志条目分割为每批1000条（OpenAI API限制）
   - 为每批生成嵌入
   - 每批完成后更新进度
   - 合并所有嵌入用于数据库插入

4. **数据库操作**：
   - 插入前截断现有表
   - 使用显式ID从1开始
   - 批量插入以优化性能
   - 插入后自动重新加载BM25索引

5. **进度跟踪**：
   - 全局 `upload_progress` 字典跟踪状态
   - `/progress` 端点提供实时状态
   - 前端通过SSE接收实时更新
   - 显示已处理/总数和耗时

### 前端功能

- **聊天界面**：实时消息显示
- **手风琴UI**：可折叠日志证据面板
- **关键词高亮**：自动高亮查询术语
- **响应式设计**：适配不同屏幕尺寸
- **文件上传UI**：拖放或点击选择
- **进度显示**：实时进度条和状态更新

## 📊 评估

### 运行评估

```bash
python evaluation/evaluate.py
```

这将：
- 从 `data/HDFS_2k.log` 加载日志
- 从 `data/benchmark/hdfs_ground_truth.json` 加载真实标签
- 运行评估并生成指标
- 保存结果到 `evaluation/results/evaluation_results.json`

### 评估指标

- **信号检测**：精确率、召回率、F1分数
- **噪声降低**：过滤掉的日志百分比（目标：95%+）
- **成本效率**：与基线方法（直接RAG）的Token使用对比
- **根因诊断**：关键词重叠、Jaccard相似度

## 🐛 故障排除

### 问题：`uvicorn` 命令未找到

**解决方案**：使用 `python -m uvicorn` 代替：
```bash
python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

### 问题：数据库连接失败

**解决方案**：
- 检查 `.env` 文件配置
- 验证Oracle wallet路径
- 确保网络可访问Oracle数据库

### 问题：OpenAI API错误

**解决方案**：
- 验证 `.env` 中的 `OPENAI_API_KEY`
- 检查API配额和账单

### 问题：未找到日志

**解决方案**：
- 通过Web界面上传日志文件（推荐）
- 或运行 `insert_logs.py` 填充数据库
- 检查数据库连接
- 验证 `docs` 表有数据

### 问题：上传失败或进度未更新

**解决方案**：
- 检查文件格式（必须是 .log, .txt, 或 .gz）
- 确保文件有有效的日志条目（每个条目至少20个字符）
- 检查文件大小（最多5000条）
- 验证 `/progress` 端点可访问
- 检查浏览器控制台错误

### 问题：CrossEncoder模型加载失败

**解决方案**：
- 确保安装了 `sentence-transformers`：`pip install sentence-transformers`
- 检查网络连接（首次运行需要下载模型）
- 查看日志文件了解详细错误信息
